# 问题
1. 在日志目录可能同时出现多个会话的日志，仅凭日志文件名很难区分是哪个会话
2. Claude Code一个会话的日志可能有多个文件，一个主日志加上若干子agent日志，目前日志选择是下拉框，很难从agent日志文件名称快速获知是哪个子Agent

# 解决方案
1. 在左侧的“日志文件”下拉框旁，增加一个按钮，弹出对话框，显示当前所有会话主日志文件列表，显示文件名、会话开始时间、会话结束时间、会话输入消息预览：
   1. 文件名：会话主日志文件名称，格式为{sessionId}.jsonl
   2. 会话开始时间：主日志文件第一条日志的timestamp时间（显示成中国时区）
   3. 会话结束时间：主日志最后一条日志的timestamp时间（显示成中国时区）
   4. 会话输入消息：主日志第一条message.role=user的日志，其中message.content数组的第一个元素，取其字符串内容作为会话输入消息，列表行显示前20个字，后面用“...”省略，鼠标悬停显示完整内容
   5. 列表仅允许单选，选择确定后，父页面日志下拉框备选项数据刷新为，选中的主日志及其子agent日志记录。
   6. 日志页面初次加载时，日志文件下拉框默认选中第一个主日志及其子agent日志
2. 日志文件下拉框中，子agent日志的选项名称应显示agent名称，而不是子agent日志文件名。agent名称通过如下步骤获取：
   1. 获取子agent日志文件名，去除后缀名".jsonl"和前缀"agent-"，仅保留agentid；
   2. 以agentid字符串，到主日志中全文搜索到第一条匹配的日志
   3. 从匹配日志递归找到tool_use_id属性，以此属性值全文搜索主日志，找到第一条匹配的日志
   4. 从匹配的日志中，递归找到subagent_type属性，此属性值为子Agent名称
